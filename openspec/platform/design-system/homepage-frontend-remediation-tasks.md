# Frontend Implementation

## Theme ownership

- [x] FE-DS-01 Remove semantic color writes from `applyTheme` so the resolved root class and data attribute are the only runtime theme selectors.
  - Blockers: DS-B01, DS-B03.
  - Primary files: `src/design-system/theme.ts`, theme tests.
  - Preserve the public `ThemeProvider`, `useTheme`, persistence, system preference, and toggle APIs.
  - Do not introduce a second palette map or component-local theme resolution.

- [x] FE-DS-02 Make first paint and hydrated theme values identical for light, dark, and system preferences.
  - Blockers: DS-B01, DS-B03.
  - Confirm that hydration does not replace stylesheet semantic values with different inline values.
  - Confirm that changing the resolved root class updates the complete semantic palette.

## Legacy aliases

- [x] FE-DS-03 Audit repository consumers of runtime `--color-*` and `--spacing-*` aliases generated by `theme.ts`.
  - Blocker: DS-B02.
  - Record every remaining consumer before removing an alias.
  - Do not treat variable definitions as consumers.

- [x] FE-DS-04 Remove unused deprecated aliases from `THEME_VARIABLES` and `getThemeCssVariables`.
  - Blocker: DS-B02.
  - If a consumer cannot be migrated in this change, document the alias, owner, consumer, and removal milestone as a compatibility exception.
  - New consumption of deprecated aliases is prohibited.

## Primary action contrast

- [x] FE-DS-05 Introduce or map an accessible semantic primary-action background using `#A8583D` with white foreground.
  - Blocker: DS-B04.
  - Required contrast: at least 4.5:1 for normal text.
  - Expected ratio for `#A8583D` and `#FFFFFF`: 5.09:1.
  - Preserve `#C46A4A` as a primitive brand value only where the applicable contrast requirement is satisfied.

- [x] FE-DS-06 Migrate Button primary state and every homepage primary action to the accessible action pair.
  - Blocker: DS-B04.
  - Verify default, hover, active, focus-visible, and disabled states.
  - Do not solve the issue by increasing label size or weight to claim large-text status.

## Shared focus and control states

- [x] FE-DS-07 Remove component-specific focus rings that conflict with the shared focus-visible contract.
  - Blocker: DS-B05.
  - Audit Button, Card, Header, Footer, ProductCard, CategoryCard, Carousel, Input, SearchInput, and contact CTA.
  - The final focus cue must remain visible on standard and inverse surfaces and achieve at least 3:1 against adjacent colors.

- [x] FE-DS-08 Replace remaining static carousel indicator colors with semantic inverse roles or a documented shared control variant.
  - Blocker: DS-B05.
  - Calculated track width, slide width, transform, loading opacity, and active-state selection may remain runtime values.
  - Static white and black color literals are not calculated runtime values.

# Acceptance Criteria

The remediation is ready for Design System re-review only when:

- [x] AC-01 No semantic palette value changes between first paint and hydrated state.
- [x] AC-02 The root class/data state is the single active theme selector.
- [x] AC-03 Runtime code does not write a partial semantic color palette.
- [x] AC-04 Deprecated aliases are removed or have an explicit temporary compatibility record.
- [x] AC-05 Primary actions with normal white text achieve at least 4.5:1.
- [x] AC-06 Focus cues achieve at least 3:1 on standard and inverse surfaces.
- [x] AC-07 Required text, status, border, and control combinations have recorded contrast evidence.
- [ ] AC-08 Homepage and Product Detail pass light/dark visual regression checks.
- [x] AC-09 TypeScript validation and the complete test suite pass.
- [x] AC-10 No business behavior, content eligibility, navigation, backend, or database behavior changes.

# Completion Handoff

## Frontend implementation evidence — 2026-07-12

- Theme runtime: `applyTheme` now changes only the root `dark` class and `data-theme`; CSS remains the sole semantic palette owner.
- Alias audit: application source has no runtime consumers of generated `--color-*` or `--spacing-*` aliases; their generator and write list were removed.
- Primary action: light and dark CSS roles match the typed `SEMANTIC_THEME_TOKENS` decision (`#A8583D` with `#FFFFFF`).
- Focus and carousel: component-local rings were removed in favor of the shared focus contract; carousel colors now use semantic inverse roles.
- Automated evidence: 19 test files and 107 tests pass, including 28 semantic contrast cases; `npx tsc --noEmit` and `npm run build` pass.
- Remaining handoff: QA-DS-01 through QA-DS-06 and AC-08 require QA visual validation and screenshots.
